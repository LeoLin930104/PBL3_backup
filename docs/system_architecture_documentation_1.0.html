Introduction

The system architecture for ShrineSeeker is divided into three tiers: presentation (frontend) tier, application (backend) tier, and database or data tier. In addition, external APIs such as Google Maps API and BestTime API are used.

Client-Server Architecture for ShrineSeeker Web Application

Client Description of Client Associated Server Description of Server
User Registration & Login Interface This interface is where the user signs up for an account or logs in to their account. It sends a http request to the User Server via the Internet. User Server This server delivers service to User Registration & Login Interface by sending a http response. It consists of the user account database which stores the user account, user name, and password.
User Page Interface This interface is where the user logs out or recovers password or checks shrines that they favorited or visited, or checks their achievements. It sends a http request to the User Server via the Internet. User Server This server delivers service to User Page Interface by sending a http response. It consists of the user account database which stores the user related information, favorited shrines, visited shrines, achievement data.
Shrine Page Interface This interface is for exploring each shrine, displaying information of a particular shrine. The user can read the shrine description, the basic information of the shrine, and view its ratings and comments. The user can also favorite or mark the shrine as visited. It sends a http request to the Shrine Display Server via the Internet. Shrines Display Server This server delivers service to Shrine Page Interface by sending a http response. It consists of the Shrine database that stores shrines and temple data such as their official names, address, opening and closing hours, built year, associated religion, and gods, entry fees, websites, and phone number, and also associated with the user account database from which fetches or stores visited and favorites.
Explore Page Interface This interface is for searching shrines by categories or view promoted shrines. It sends a http request to the Shrines Search Server via the Internet. Shrines Search Server This server delivers service to Explore Page Interface by sending a http response. It consists of the Shrine database to access the list of shrines and temples. 
Map Page Interface This interface is for viewing the map and requesting for shrine hopping routes. It sends a http request to the Route Recommendation Server via the Internet. Route Recommendation Server This server delivers service to Map Page Interface by sending a http response. It consists of the Shrine Database to display nearby shrines and temples with crowdedness data obtained from BestTime API. It also accesses the user’s preferences data from the user account database to recommend a route while obtaining routing to destination data from Google Maps API.

Pipe and Filter Architecture Diagram

The user starts with signing in by inputting a new email and new password that will be stored in the user database. When the existing user logs in, the inputted existing email and existing password will be validated by the data from the user database and returned so that the user can log in. If the user wants to recover their forgotten password, a new password will be stored in the user database after going through the recovery password procedure.

After user registration is done, the user can access the map page that shows the area around the user fetched from Google Maps API. The nearest shrines and crowdedness level are displayed by data obtained from the Shrine database and BestTime API respectively.

The recommendation system is based on the user’s preferences stored in the user database. In addition to the user’s personal information, the user database stores shrines that have been searched and those favorited by the user. After processing the data, the recommendation system produces the routes.

Map Page Interface The interface that visualizes the map around the user with the nearest shrines and their respective crowdedness level

Shrine Information Display Service The service that displays shrine information stored in the shrine database
Shrine Search Service The service that allows users to search the shrines by category or name, and view promoted shrines
User Database The database that stores the personal information and preferences referenced from history and favorite shrines
Shrine Database The database that stores the shrines information, including the name, address, operating hours, built year and etc
Google map API The API to show the map in the Map Interface and display route to destination in recommending shrines
BestTime API The API to obtain the crowdedness data of the shrines
