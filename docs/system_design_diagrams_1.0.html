User Page Activity Diagram

The activity diagram describes the progress of logging out, deleting the account, checking achievements and checking shrines that are favorited and visited. All of these functions are available on the user page. One thing to note is that the user has to be logged in before doing the above mentioned functions.

Shrine Page Activity Diagram

Implemented four use cases, including check/leave comments, check/leave ratings, favorite, and visited. They will all be on every shrine page.

Map Page Activity Diagram

This diagram describes the process involved in the recommendation system on the map. The route recommendation system is based on what has accumulated in the user's preferences. If the user is new, the system will use the default preference. The user will be recommended two routes; main route and secondary route and the user will have to choose either one. After this, the route is considered to be chosen and the system will display the detailed information of that route. If the user does not like the recommended routes, there is an option to regenerate the routes. The map icons of nearby shrines as well as their crowdedness level. When the icon of a particular shrine is clicked, it leads to its respective shrine page.

Account Creation Activity Diagram

This activity diagram shows the signing up procedure for users to create an account. The requirement of the password should be 8 characters long including both uppercase and lowercase letters. Users can also retry to create an account if the link has expired.

Login Activity Diagram

The process of login to reach the user's homepage with processing email and passwords. It also guides users to sign up and forget the password page if they do not have an account or have forgotten passwords.

Looking up Shrine by Name or Category Sequence Diagram

This sequence diagram shows the flow in which the user searches for a particular shrine information. There are two types of search method, name and category.

Sequence Diagram for "Request for Shrine Hopping Routes"

This sequence diagram describes the process involved in requesting shrine hopping routes. The routes requested by the system are obtained from Google Maps API and displayed to the user in the user interface.

Sequence Diagram for "Check Crowdedness Around Shrines"

This sequence diagram describes the process in checking the crowdedness around the shrines. The user should be able to see the crowdedness level near each shrine, in which the data is retrieved from the BestTime API.

Sequence Diagram for "Recover Password"

This sequence diagram shows the process for recovering the password if a user has forgotten the password to their account. The link to reset password will expire within 10 minutes and the user will have to try again if it expires.

Sequence Diagram for "Add Shrine"

This sequence diagram shows the flow of shrine being added by the administrator. This is done in cases where new shrines have been built or shrines that are not already in our database are added.
